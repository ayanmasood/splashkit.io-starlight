:::tip[SplashKit Database allows you to create, query and otherwise manipulate a database. ]
SplashKit's database library allows you to easily create or load databases and to perform queries on them. To get started with databases in SplashKit, the first thinkg you need to do is to load or create a database file. You can do this by calling the `Open Database(string Name)` function.

## Open Database [](#open-database)

 
Loads or creates, and returns a database.

The supplied `filename` is the name of the file of the database. The
supplied `name` indicates the name to use to refer to this [`Database`](/components/database/#database).
The [`Database`](/components/database/#database) can then be retrieved by passing this `name` to
the `database_named` function. If the database file does not exist, it
will be created for you.

**Return Type**

- [`Database`](/components/database/#database) 

**Parameters**

| Name   | Type                                               | Description                                                                        |
| ------ | -------------------------------------------------- | ---------------------------------------------------------------------------------- |
| name | `String` | The name used to refer to the database. |
| filename | `String` | The filename used to locate the database to use. |

**Signatures**

## TYPES
### Database [](#database)
The [`Database`](/components/database/#database) type is used to refer to databases that can be manipulated by the SplashKit database code. Databases are:

- opened with [Open Database](/components/database/#open-database),

- and must be released using [Free Database](/components/database/#free-database) (to release a specific database) or `Delete All Databases` (to release all loaded databases).

### Query Result [](#query-result)

The [`Query Result`](/components/database/#query-result) type is used to store the result of performing a query on the database. It can:

- Show the success or failure of a query using [`Query Success`](/components/database/#query-success).

- Store the data from queries that return data. This data is accessed using [`Query Column For Boolean`](/components/database/#query-column-for-boolean), [`Query Column For Integer`](/components/database/#query-column-for-integer), [`Query Column For String`](/components/database/#query-column-for-string), and [`Query Column For Double`](/components/database/#query-column-for-double).
